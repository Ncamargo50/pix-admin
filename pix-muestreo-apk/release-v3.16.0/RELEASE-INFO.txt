Why is this SMALLER than v3.14.0 (6.50 MB)?
  v3.14.0 shipped 7,290 Java classes (classes.dex = 8.5 MB), including
  ~2,200 Guava classes, ~360 Kotlin coroutine flow classes, ~260 AppCompat
  widget classes — none of which a TWA actually invokes. R8 now strips that
  dead code. After the fix pass, classes.dex is 1.5 MB with ALL the classes
  that ARE called preserved by-name: Play Integrity (IntegrityManager,
  IntegrityManagerFactory, IntegrityTokenRequest/Response, Builder inners),
  androidx.biometric (BiometricFragment + BiometricPrompt lifecycle),
  GMS Tasks (OnSuccessListener/OnFailureListener), androidbrowserhelper
  (TWA/CustomTabs runtime), and our 4 bridges (OfflineActivity,
  BiometricBridge, IntegrityBridge, GNSSBridge) with @JavascriptInterface
  method names intact.

CHANGES v3.15.0 → v3.16.0 (the 9.5/10 hardening pack)
------------------------------------------------------
  1. Sentry crash reporting
       • app/src/main/assets/js/telemetry.js  (new)
       • PII-scrubbed, opt-out via setting, lazy CDN load
       • Fed by auth.js (clearUser on logout) + agent-field.js (global
         error + unhandledrejection handlers)

  2. AES-GCM 256 at-rest encryption for IndexedDB
       • app/src/main/assets/js/crypto-vault.js  (new)
       • PBKDF2-SHA256, 250k iterations, 32-byte random salt per install
       • Envelope: { __enc:'pix-aesgcm-v1', v:1, iv:b64, ct:b64, t:'s'|'b' }
       • Wired into db.js saveFile / getFileAsBlob — transparent for callers
       • Vault unlocks automatically in app.js bootstrap with device secret;
         legacy plaintext rows still read.

  3. Biometric gate (Class 3 STRONG + device credential fallback)
       • app/src/main/assets/js/biometric.js  (new JS wrapper)
       • app/src/main/java/.../BiometricBridge.java  (new)
       • Fires after master-key password verification in auth.js; user cancel
         counts as a failed attempt to prevent "cancel-to-bypass".
       • Gracefully no-ops on devices without a sensor (isAvailable → no-hardware).

  4. APK Signature Scheme v3 + v4
       • app/build.gradle: enableV3Signing=true, enableV4Signing=true
       • v3 gives us key-rotation capability and satisfies Play Store's 2027
         baseline. v4 enables incremental install on Android 11+.
